# Checkpoint

Write a structured checkpoint to project memory so the next session can resume smoothly. Use the `memory` tool to write a file named `checkpoint.md` with the following sections:

## Format

```
# Checkpoint

## Goal
<the current goal or task, in one sentence>

## Current Task
<what you are working on right now>

## Progress
<bullet list of what is done>

## Blockers
<bullet list of what is blocking progress, or "none">

## Next Steps
<bullet list of the immediate next actions>
```

## Rules

- Keep it concise. This is a resume aid, not a full report.
- Overwrite any existing `checkpoint.md` (use `memory write checkpoint.md`).
- Focus on what the next session needs to know to continue.
